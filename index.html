Senior Design Project (CprE 491)

3D Reconstruction using Time of Flight Sensors

This semester, Spring 15, I began my senior design project. It is in the earliest stages, and this section will be updated as the semester goes on. The client for this project is VirtuSense Technologies.

The project description is as follows:
VirtuSense Technologies (VST) intends to explore time-of-flight based technology that resolves distance (depth) based on the known speed of light for 3D surface reconstruction. VST is further interested in developing algorithms to morph shapes of 3D models, using force fields.

I am the webmaster for the project's website and you can find the project's website linked above!

The project's goal is to introduce this 3D technology into the field of plastic surgery. The idea is to create an inexpensive way to take a picture of someone and allow a surgeon to show them on the potential changes on a 3D replica of the face. This can provide an avenue for more input from the patient or even a piece of mind.

Basic UNIX Shell (CprE 308)

Computer Engineering 308 is Operating Systems, Principles and Practices, and probably one of my favorite computer classes at Iowa State. The class centers around teaching about operating systems using UNIX. Not only did I learn about the purpose of an operating system, processes and threads, process scheduling, and deadlocks, but I got to program my own UNIX shell.

My experience with C began in the spring of 2012, and that was about all I got. This class allowed me to not only dive deeper into C and use it to make something cool. My shell accepted basic commands and prompts, and created parent and child processes. This project involved a lot of StackOverflow and various resources from other universities (Harvard and Stanford had a lot on the subject) seeing as I knew nothing when I started. But the end result was extremely satisfying and I learned a metric ton of things.

Measuring and Studying Run Times (ComS 311)

Computer Science 311 is the design and analysis of algorithms. Many might consider it to be a personal hell. But nevertheless, extremely important. As the title of the course suggests, the project was the analysis of run times of certain algorithms.

QuickSort, InsertionSort and MergeSort were all used in this project, utilizing the Java programming language. The goal was to create arrays of random data and find the largest array size that would yield a run time more than one second. I made worst case and average case running times for each algorithm. To analyze these running times, the size and times for each were graphed and discussed. This process allowed me to really look in depth at each of these sorting algorithms, and their individual strengths and weaknesses. (I hate to admit this, but I actually enjoyed this project quite a lot).
